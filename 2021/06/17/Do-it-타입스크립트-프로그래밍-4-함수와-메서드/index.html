함수 선언문과 함수 표현식의 차이점

  • 함수 선언문에서는 호이스팅 현상이 생기지만, 함수 표현식에서는 호이스팅 현상이 생기지 않는다.
  • 하지만 strict mode를 사용하면 함수가 선언되기 전에 해당 함수를 호출하면 오류가 발생하기 때문에 두 방식의 차이점은 없어지는 거 같다(?)

선택적 매개변수

  • 인자를 전달해주지 않아도 되는 매개변수.
  • 인터페이스의 선택 속성을 만들 때와 비슷하게 매개변수 이름 뒤에 ?를 붙이면 선택적 매개변수를 만들 수 있다.
  • 인자를 전달해주지 않으면 매개변수의 값은 undefined가 된다.
  • 함수의 매개변수 순서상, 선택적 매개변수 다음에 필수 매개변수가 올 수 없다.
  • 선택적 매개변수 다음에 필수 매개변수가 오면, 인자가 어떤 매개변수의 값인지 구분할 수 없기 때문이다.

매개변수 기본값 지정하기

  • 인자가 전달되지 않았을 때 매개변수에 대신 전달될 기본값을 지정할 수 있다.

함수 시그니처

  • 함수의 형태를 타입으로 지정하는 것.
  • type 키워드로 함수 시그니처에 이름을 붙일 수 있다.

실행문과 표현식의 차이

실행문(statement)

  • 어떤 동작을 실행하는 코드 조각.
  • 실행문의 종류는 여기에서 확인 가능.
  • 🌟 기본적으로 값을 반환하지 않는다 🌟
  • 함수의 몸통에는 return 문을 사용해서 값을 반환하게 할 수 있다.
  • (자바스크립트에서 모든 실행문은 결과 값을 가진다고 한다…)

표현식(expression)

  • 값으로 결정(또는 계산)될 수 있는 유효한 코드 조각.
  • 보통 표현식은 리터럴, 연산자, 변수, 함수 호출 등이 복합적으로 구성되어 있다.
  • 표현식의 종류는 여기에서 확인 가능.
  • 🌟 값을 반환한다 🌟
  • (비교 연산자라면 boolean 값, 산술 연산자라면 number 값을 반환함)
  • 값이 위치할 것이라고 예상되는 곳에 표현식을 대신 넣을 수 있다.
  • (예를 들면, 함수의 인자나 대입 연산자의 오른쪽 부분 등)

표현식 문(expression statement)

  • 실행문이 위치할 것이라고 예상되는 곳에 표현식을 넣는 것.
