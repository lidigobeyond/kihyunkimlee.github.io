[Do it! 타입스크립트 프로그래밍] 3. 객체와 타입

작성일 2021-06-15 Edited on 2021-06-17 In Do it! 타입스크립트 프로그래밍 Views: Disqus:

아래의 내용은 전예홍님의 <Do it! 타입스크립트 프로그래밍> 책을 공부하며 중요한 내용을 정리한 것입니다.

var와 let&const의 차이

var

  • 함수 스코프 사용.
  • var 변수를 재정의해도 오류가 발생하지 않음.
  • 호이스팅이 일어남.(var 변수가 정의되기 전에도 접근하여 사용 가능)

let과 const

  • 블록 스코프 사용.
  • 재정의하면 오류 발생.(const는 재할당해도 오류가 발생함)
  • 호이스팅이 일어나지 않음.

타입주석과 타입 추론

타입 주석

  • 변수의 타입을 명시하는 것.
  • 변수의 타입이 정해지면, 변수는 정해진 타입에 해당하는 값만 가질 수 있다.

타입 추론

  • 타입 주석 부분을 생략하였을 때 컴파일러가 변수에 할당되는 초깃값에 따라서 변수의 타입을 지정하는 것.

타입

타입스크립트 기본 타입

  • number
  • boolean
  • string
  • object

any 타입과 undefined 타입

any 타입

  • 어떤 타입의 값도 저장할 수 있는 타입.
  • 자바스크립트와 호환을 위해서 제공되는 타입.
  • 타입 상속 관계(?)에서 any 타입은 최상위 타입이다.

undefined 타입

  • undefined 값만 저장할 수 있는 타입.
  • 자바스크립트에서 undefined는 값이었지만, 타입스크립트에서 undefined는 값이자 타입이다.
  • 타입 상속 관계에서 undefined 타입은 최하위 타입이다.

객체와 인터페이스

object 타입

  • object 타입은 마치 객체를 대상으로 하는 any 타입처럼 동작한다.
  • object 타입 변수에 number, boolean, string 타입의 값을 저장할 수 없지만, 어떠한 형태의 객체도 저장할 수 있다.

인터페이스

  • 어떤 변수에 저장될 객체의 형태를 제한시킬 때 사용하는 문법.
  • 인터페이스 문법을 통해서 객체의 특정한 형태를 타입으로 정의할 수 있다.
  • 타입에는 해당 타입의 객체가 가져야 할 속성과 그 속성의 타입을 명시할 수 있다.

선택 속성

  • 가지고 있어도 되고 안 가지고 있어도 되는 성질.
  • 속성 이름 뒤에 ? 를 붙이면 해당 속성은 선택 속성이 된다.

익명 인터페이스

  • 이름이 없는 인터페이스.
  • interface 키워드를 사용하지 않는다.
  • 주로 한 번만 사용될 인터페이스를 만들 때 익명 인터페이스로 정의한다.

클래스

클래스 선언문

  • 타입스크립트의 클래스 선언문은 다음과 같다.

접근 제한자

  • 접근 제한자에는 public, private, protected이 있다.

    • public
      • 자식 클래스에서 접근 가능 ⭕
      • 클래스 외부에서도 접근 가능 ⭕
    • protected
      • 자식 클래스에서 접근 가능 ⭕
      • 클래스 외부에서는 접근 불가능 ❌
    • private
      • 자식 클래스에서 접근 불가능 ❌
      • 클래스 외부에서도 접근 불가능 ❌

  • 접근 제한자를 생략하면 모두 public 속성으로 간주한다.

생성자

  • constructor라는 이름의 특수한 메서드.
  • 객체가 생성될 때 객체를 초기화해준다.
  • 생성자의 매개변수에 접근 제어자를 붙이면 해당 매개변수의 이름을 가진 속성이 클래스에 선언된 것처럼 동작한다.
  • 그러므로 아래의 Employee1 클래스와 Employee2 클래스는 완전히 동일하게 동작한다.
  • 어떤 클래스가 다른 클래스를 상속했을 경우, super 키워드를 통해서 부모 클래스의 생성자를 호출할 수 있다.

인터페이스

  • 다른 객체 지향 언어와 마찬가지로 타입스크립트 클래스도 인터페이스를 구현할 수 있다.
  • 클래스가 인터페이스를 구현할 때는 아래와 같이 implements 키워드를 사용한다.

추상 클래스

  • 완전하게 구현되어 있지 않은 메서드, 즉 추상 메서드를 가지고 있는 클래스.
  • 추상 클래스는 class 키워드 앞에 abstract 키워드를 붙여서 만든다.

추상 메서드

  • 부모 클래스를 상속하는 자식 클래스들이 공통으로 가져야 할 메서드이지만, 메서드의 구체적인 동작을 부모 클래스가 미리 정의하기 어려울 때 해당 메서드를 추상 메서드로 정의한다.
  • 추상 메서드는 메서드 앞에 abstract 키워드를 붙여서 만든다.
  • 추상 클래스를 상속하는 클래스는 추상 메서드를 반드시 구현해야 한다.
  • (추상 속성은 뭐지…?)

클래스 상속

  • extends 키워드를 사용해서 상속 클래스를 만든다.

static 속성

  • 클래스의 모든 인스턴스가 값을 공유하는 속성.
  • static 속성은 속성명 앞에 static 키워드를 붙여서 만든다.
  • 클래스의 static 속성은 ‘클래스명 . static 속성명‘ 같이 점 표기법을 사용해서 접근할 수 있다.

비 구조화 할당

비 구조화 할당

  • 객체 안에 있는 속성을 추출해서 변수 또는 상수로 바로 선언해줄 수 있는 ES6 표현식.
  • 타입스크립트에서는 비 구조화 할당을 할 때 기본값을 설정해줄 수 없는 거 같다(?)
  • 비 구조화 할당 시 변수 또는 상수의 이름을 바꿔주는 건 가능하다.

잔여(rest) 연산자

  • 비 구조화 할당에서 특정 속성을 제외한 나머지 속성들을 별도의 변수 또는 상수에 한꺼번에 할당하고 싶을 때 사용하는 연산자.

전개(spread) 연산자

  • 객체의 모든 속성을 새롭게 생성되는 객체의 속성으로 집어넣고 싶을 때 사용하는 연산자.

타입 단언

타입 단언

  • 컴파일러가 가진 변수의 타입 정보를 무시하고 프로그래머가 정해준 임의의 타입으로 해석하도록 컴파일러에 지시하는 것.
  • 또는 컴파일러가 실제 런타임에 존재할 변수의 타입과 다르게 추론하거나 너무 보수적으로 변수의 타입을 추론할 경우, 프로그래머가 컴파일러에게 변수의 타입에 대한 힌트를 주는 것.
  • 타입 단언은 아래와 같이 두 가지 형태로 만들 수 있다.
    • (<타입명>객체명)
    • (객체명 as 타입명)
