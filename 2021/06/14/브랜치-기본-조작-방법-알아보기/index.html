브랜치 기본 조작 방법 알아보기

작성일 2021-06-14 Edited on 2021-06-15 In Git Views: Disqus:

브랜치

  • 커밋 사이를 가볍게 이동할 수 있는 어떤 포인터 같은 것(?)
  • 브랜치를 사용하면 여러가지 작업을 서로 영향을 주지 않으면서 동시에 수행할 수 있다.

master 브랜치

  • 첫 커밋을 만들면 Git이 자동으로 만드는 브랜치.
  • 커밋을 만들 때마다 자동으로 가장 마지막 커밋을 가리킨다.

HEAD 포인터

  • 현재 작업중인 브랜치를 가리키는 포인터.
  • 기본적으로 master 브랜치를 가리키고 있다.

브랜치 기본 조작 방법

브랜치 생성하기

  • git branch 명령으로 새로운 브랜치를 생성할 수 있다.
  • 새롭게 생성된 브랜치는 자동으로 지금 작업하고 있는 커밋을 가리킨다.

브랜치 목록 보기

  • git branch 명령으로 브랜치 목록을 볼 수 있다.

브랜치 이동하기

  • git checkout 명령으로 다른 브랜치로 이동할 수 있다.
  • 브랜치를 이동한다는 것은 HEAD 포인터가 다른 브랜치를 가리키도록 수정한다는 것이다.
  • 브랜치를 이동하면 워킹 디렉터리의 파일이 변경된다.
    • 브랜치를 이동하면 워킹 디렉터리의 파일이 해당 브랜치가 가리키는 커밋의 스냅샷으로 변경된다.
    • 단, 마지막 커밋 이후로 변경 사항이 존재하는 브랜치로 이동할 경우, 워킹 디렉터리의 파일은 그 브랜치에서 가장 마지막으로 했던 작업 내용으로 변경된다.
  • 현재 브랜치에 아직 커밋하지 않은 파일이 이동하려는 브랜치에 있는 파일과 충돌된다면, 브랜치 이동을 할 수 없다!
    • 이럴 때는 현재 브랜치의 워킹 디렉터리를 정리하거나 git stach 명령을 사용해야 한다.

브랜치 삭제하기

  • git branch 명령에 -d 옵션을 주어서 브랜치를 삭제할 수 있다.
  • 아직 머지하지 않은 커밋을 갖고 있는 브랜치를 강제로 삭제할 때는 -D 옵션을 사용한다.
